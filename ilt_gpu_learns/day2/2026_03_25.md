### 1.2 CUDA线程层次结构

#### 生活中的类比：工厂与工人

想象一个大型工厂（GPU）：
- **Grid（网格）** = 整个工厂
- **Block（线程块）** = 车间
- **Thread（线程）** = 工人
- **Warp（线程束）** = 工作小组（32人一组）

**工作流程**：
1. 工厂接到任务（启动核函数）
2. 将任务分配给多个车间（Block）
3. 每个车间有多个工人（Thread）
4. 工人按工作小组（Warp）协同工作

**关键规则**：
- 每个车间（Block）独立工作，不能直接通信
- 车间内的工人可以共享资源（共享内存）
- 工作小组（Warp）是调度的基本单位

#### 线程索引计算

**一维网格**：
```
全局线程索引 = blockIdx.x * blockDim.x + threadIdx.x
```

**例子**：
- Block 0: 线程 0-255（假设blockDim.x=256）
- Block 1: 线程 256-511
- Block 2: 线程 512-767
- ...

**二维网格**（处理图像常用）：
```cpp
int x = blockIdx.x * blockDim.x + threadIdx.x;
int y = blockIdx.y * blockDim.y + threadIdx.y;
int index = y * width + x;  // 行优先存储
```

### 1.3 Warp（线程束）机制

#### 什么是Warp？

**Warp** 是GPU执行的基本单位，包含**32个连续线程**。

**关键特性**：
1. **SIMT执行**：同一Warp中的32个线程同时执行相同指令
2. **锁步执行**：所有线程同步前进
3. **分支分歧**：同一Warp中线程走不同分支会导致串行执行

#### 生活中的类比：合唱团

想象一个32人的合唱团（Warp）：
- 指挥（GPU调度器）给出指令
- 所有人同时唱同一句（SIMT）
- 如果有人唱高音，有人唱低音（分支分歧），只能分两次唱

#### 分支分歧问题

**问题代码**：
```cpp
__global__ void branchDivergence(int *data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        if (data[idx] > 0) {
            // 路径A：约一半线程走这里
            data[idx] = data[idx] * 2;
        } else {
            // 路径B：约一半线程走这里
            data[idx] = data[idx] + 1;
        }
    }
}
```

**执行过程**：
1. Warp中一半线程走路径A，一半走路径B
2. GPU先执行路径A，另一半线程空闲等待
3. 再执行路径B，前一半线程空闲等待
4. **性能损失约50%**

**优化方法**：重新组织数据，使同一Warp的线程走相同路径

```cpp
__global__ void noBranchDivergence(int *positive, int *negative, 
                                    int n_pos, int n_neg) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    
    // 所有线程要么处理positive，要么处理negative
    if (idx < n_pos) {
        positive[idx] = positive[idx] * 2;
    }
    if (idx < n_neg) {
        negative[idx] = negative[idx] + 1;
    }
}
```

### 1.4 CUDA核函数执行模型

#### GPU硬件架构：SM（Streaming Multiprocessor）

```
GPU
├── SM 0
│   ├── CUDA Core 0, 1, 2, ..., 63  (执行计算)
│   ├── Warp Scheduler 0, 1         (调度Warp)
│   ├── Shared Memory               (96KB)
│   └── Register File               (256KB)
├── SM 1
│   └── ...
├── ...
└── Global Memory (HBM2, 24GB for RTX 3090)
```

**执行流程**：
1. GPU将Block分配给SM
2. SM将Block中的线程分成Warp（每32个线程）
3. Warp Scheduler调度Warp执行
4. 当一个Warp等待内存访问时，切换到另一个Warp（延迟隐藏）

#### Block与SM的关系

**规则**：
- 一个Block只能在一个SM上执行
- 一个SM可以同时执行多个Block
- SM资源限制：寄存器、共享内存、线程数

**RTX 3090的SM限制**：
- 每个SM最多1536个线程
- 每个SM最多16个Block
- 每个Block最多1024个线程
- 每个SM 256KB寄存器、100KB共享内存

#### 占用率（Occupancy）

**占用率** = 活跃Warp数 / SM最大Warp数

**计算示例**：
```
假设：
- blockDim = 256线程/块 = 8 Warp/块
- 每个SM最多48 Warp
- 理论最多6个Block/SM
- 但受寄存器和共享内存限制
```

**优化目标**：保持占用率在50%以上，以隐藏延迟

## 第二部分：GPU内存层次与优化 (下午 14:00-16:00)

### 2.1 GPU内存层次结构

#### 内存类型对比

| 内存类型 | 位置 | 访问速度 | 容量 | 作用域 |
|---------|------|----------|------|--------|
| **寄存器** | GPU芯片 | 1 cycle | 几KB | 线程私有 |
| **共享内存** | GPU芯片 | ~25 cycles | 48-100KB | 块内共享 |
| **L1缓存** | GPU芯片 | ~30 cycles | 128KB/SM | SM私有 |
| **L2缓存** | GPU芯片 | ~100 cycles | 6MB | 全局共享 |
| **全局内存** | HBM2 | ~500 cycles | 24GB | 全局 |

**关键洞察**：
- 寄存器最快，但数量有限
- 共享内存比全局内存快20倍
- 全局内存带宽最高（936GB/s for RTX 3090）

#### 内存访问模式

**合并访问（Coalesced Access）**：
- 同一Warp的32个线程访问连续的32个字（128字节）
- 一次内存事务完成，效率最高

**非合并访问**：
- 线程访问不连续地址
- 多次内存事务，性能下降

**例子**：

```cpp
// ✓ 合并访问 - 高效
__global__ void coalescedAccess(float *data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        data[idx] = idx;  // 连续访问
    }
}

// ✗ 非合并访问 - 低效
__global__ void stridedAccess(float *data, int n, int stride) {
    int idx = (blockIdx.x * blockDim.x + threadIdx.x) * stride;
    if (idx < n) {
        data[idx] = idx;  // 跨步访问，间隔stride
    }
}
```

### 2.2 共享内存（Shared Memory）

#### 什么是共享内存？

**共享内存**是GPU上的高速缓存，位于SM内部，供同一Block内的线程共享。

**特点**：
- 延迟低：约25 cycles（全局内存500 cycles）
- 带宽高：数TB/s
- 可编程：程序员显式管理
- 容量小：RTX 3090每SM 100KB

#### 共享内存的Bank冲突

**Bank结构**：共享内存分为32个Bank，每个Bank宽度4字节

**访问规则**：
- 不同线程访问不同Bank → 并行访问，最快
- 多个线程访问同一Bank → 串行访问，Bank冲突

**例子**：

```cpp
// ✓ 无Bank冲突
__shared__ float data[256];
data[threadIdx.x] = threadIdx.x;  // 每个线程访问不同Bank

// ✗ Bank冲突（2路冲突）
__shared__ float data[256];
data[threadIdx.x * 32] = threadIdx.x;  // 每32个线程访问同一Bank

// ✓ 解决方案：填充
__shared__ float data[256 + 1];  // +1避免冲突
data[threadIdx.x * 33] = threadIdx.x;
```

#### 共享内存使用示例

创建文件`shared_memory_demo.cu`：

```cpp
/*
 * Day 2 - 共享内存示例
 * 
 * 目标：理解共享内存的使用和性能优势
 * 
 * 学习要点：
 * 1. 共享内存的声明和使用
 * 2. Bank冲突的避免
 * 3. 数据共享和同步
 */

#include <cuda_runtime.h>
#include <stdio.h>
#include <stdlib.h>

// 全局内存版本：简单归约求和
__global__ void reduceGlobal(float *input, float *output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    
    // 每个线程独立计算部分和
    float sum = 0.0f;
    for (int i = idx; i < n; i += blockDim.x * gridDim.x) {
        sum += input[i];
    }
    
    // 使用原子操作累加（性能较差）
    atomicAdd(output, sum);
}

// 共享内存版本：并行归约求和
__global__ void reduceShared(float *input, float *output, int n) {
    // 声明共享内存（动态分配）
    extern __shared__ float sdata[];
    
    int tid = threadIdx.x;
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    
    // 第一步：每个线程加载数据到共享内存
    sdata[tid] = 0.0f;
    if (idx < n) {
        sdata[tid] = input[idx];
    }
    __syncthreads();  // 同步：确保所有数据已加载
    
    // 第二步：并行归约
    // 每次迭代，线程数减半
    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) {
            sdata[tid] += sdata[tid + s];
        }
        __syncthreads();  // 同步：确保这一轮完成
    }
    
    // 第三步：Block 0的线程0写入结果
    if (tid == 0) {
        atomicAdd(output, sdata[0]);
    }
}

// 共享内存示例：矩阵转置
__global__ void transposeShared(float *input, float *output, 
                                  int width, int height) {
    // 声明共享内存（静态分配）
    __shared__ float tile[16][16];  // 16x16块
    
    // 坐标计算
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    
    // 从全局内存加载到共享内存
    // 注意：这里用到了矩阵分块的思想
    if (x < width && y < height) {
        tile[threadIdx.y][threadIdx.x] = input[y * width + x];
    }
    __syncthreads();
    
    // 计算转置后的坐标
    int x_t = blockIdx.y * blockDim.y + threadIdx.x;
    int y_t = blockIdx.x * blockDim.x + threadIdx.y;
    
    // 从共享内存写回全局内存
    if (x_t < height && y_t < width) {
        output[y_t * height + x_t] = tile[threadIdx.x][threadIdx.y];
    }
}

// 测试归约性能

    }

    cudaMalloc(&d_input, N * sizeof(float));
    cudaMalloc(&d_output, sizeof(float));
    cudaMemcpy(d_input, h_input, N * sizeof(float), cudaMemcpyHostToDevice);
    
    // 创建CUDA事件用于计时
    cudaEvent_t start, stop;
    cudaEventCreate(&start);
    cudaEventCreate(&stop);
    float time;
    
    // 测试全局内存版本
    cudaMemset(d_output, 0, sizeof(float));

    reduceGlobal<<<numBlocks, blockSize>>>(d_input, d_output, N);
    cudaEventRecord(stop);
```

### 3.1 矩阵乘法：GPU编程的经典案例

矩阵乘法是GPU优化的经典问题，能够全面展示：
- 线程组织
- 内存访问优化
- 共享内存应用
- 性能分析和调优

#### 矩阵乘法公式

```
C = A × B

其中:
C[i][j] = Σ(k=0 to K-1) A[i][k] × B[k][j]

假设:
- A: M × K 矩阵
- B: K × N 矩阵
- C: M × N 矩阵
```

### 3.2 实现版本对比

我们将实现4个版本，逐步优化：

1. **CPU版本**：基准对比
2. **GPU朴素版本**：直接并行化
3. **GPU优化版本1**：使用共享内存
4. **GPU优化版本2**：分块+共享内存

创建文件`matrix_multiplication.cu`：

```cpp
/*
 * Day 2 - 矩阵乘法优化
 * 
 * 目标：通过矩阵乘法掌握GPU性能优化技巧
 * 
 * 学习要点：
 * 1. CPU vs GPU性能对比
 * 2. 全局内存访问优化
 * 3. 共享内存使用
 * 4. 分块（Tiling）技术
 */

#include <cuda_runtime.h>
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

// 矩阵尺寸
#define M 1024
#define K 1024
#define N 1024

// CPU版本：基准
void matmulCPU(float *A, float *B, float *C, int m, int k, int n) {
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            float sum = 0.0f;
            for (int l = 0; l < k; l++) {
                sum += A[i * k + l] * B[l * n + j];
            }
            C[i * n + j] = sum;
        }
    }
}

// GPU版本1：朴素实现
__global__ void matmulNaive(float *A, float *B, float *C, 
                             int m, int k, int n) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    
    if (row < m && col < n) {
        float sum = 0.0f;
        for (int l = 0; l < k; l++) {
            sum += A[row * k + l] * B[l * n + col];
        }
        C[row * n + col] = sum;
    }
}

// GPU版本2：使用共享内存（分块矩阵乘法）
#define TILE_SIZE 32

__global__ void matmulShared(float *A, float *B, float *C,
                              int m, int k, int n) {
    // 共享内存块
    __shared__ float As[TILE_SIZE][TILE_SIZE];
    __shared__ float Bs[TILE_SIZE][TILE_SIZE];
    
    // 计算全局坐标
    int row = blockIdx.y * TILE_SIZE + threadIdx.y;
    int col = blockIdx.x * TILE_SIZE + threadIdx.x;
    
    float sum = 0.0f;
    
    // 分块循环
    for (int t = 0; t < (k + TILE_SIZE - 1) / TILE_SIZE; t++) {
        // 加载数据到共享内存
        int aRow = row;
        int aCol = t * TILE_SIZE + threadIdx.x;
        int bRow = t * TILE_SIZE + threadIdx.y;
        int bCol = col;
        
        // 边界检查
        As[threadIdx.y][threadIdx.x] = (aRow < m && aCol < k) ? 
                                        A[aRow * k + aCol] : 0.0f;
        Bs[threadIdx.y][threadIdx.x] = (bRow < k && bCol < n) ? 
                                        B[bRow * n + bCol] : 0.0f;
        
        __syncthreads();  // 确保数据已加载
        
        // 计算部分积
        for (int l = 0; l < TILE_SIZE; l++) {
            sum += As[threadIdx.y][l] * Bs[l][threadIdx.x];
        }
        
        __syncthreads();  // 确保计算完成
    }
    
    // 写回结果
    if (row < m && col < n) {
        C[row * n + col] = sum;
    }
}

// 初始化矩阵
void initMatrix(float *mat, int rows, int cols, float value) {
    for (int i = 0; i < rows * cols; i++) {
        mat[i] = value;
    }
}

// 验证结果
bool verifyResult(float *C1, float *C2, int m, int n, float epsilon = 1e-3) {
    for (int i = 0; i < m * n; i++) {
        if (fabs(C1[i] - C2[i]) > epsilon) {
            printf("错误: index %d, 值 %.6f vs %.6f\n", 
                   i, C1[i], C2[i]);
            return false;
        }
    }
    return true;
}

// 性能测试
void benchmark() {
    printf("\n========================================\n");
    printf("矩阵乘法性能测试\n");
    printf("矩阵尺寸: A(%dx%d) × B(%dx%d) = C(%dx%d)\n", M, K, K, N, M, N);
    printf("========================================\n\n");
    
    // 分配主机内存
    float *h_A = (float*)malloc(M * K * sizeof(float));
    float *h_B = (float*)malloc(K * N * sizeof(float));
    float *h_C_cpu = (float*)malloc(M * N * sizeof(float));
    float *h_C_naive = (float*)malloc(M * N * sizeof(float));
    float *h_C_shared = (float*)malloc(M * N * sizeof(float));
    
    // 初始化数据
    printf("初始化矩阵...\n");
    initMatrix(h_A, M, K, 1.0f);
    initMatrix(h_B, K, N, 1.0f);
    
    // 分配设备内存
    float *d_A, *d_B, *d_C;
    cudaMalloc(&d_A, M * K * sizeof(float));
    cudaMalloc(&d_B, K * N * sizeof(float));
    cudaMalloc(&d_C, M * N * sizeof(float));
    
    // 拷贝数据到GPU
    cudaMemcpy(d_A, h_A, M * K * sizeof(float), cudaMemcpyHostToDevice);
    cudaMemcpy(d_B, h_B, K * N * sizeof(float), cudaMemcpyHostToDevice);
    
    // 创建计时事件
    cudaEvent_t start, stop;
    cudaEventCreate(&start);
    cudaEventCreate(&stop);
    float time;
    
    // ===== CPU版本 =====
    printf("\n[1] CPU版本\n");
    clock_t cpu_start = clock();
    matmulCPU(h_A, h_B, h_C_cpu, M, K, N);
    clock_t cpu_end = clock();
    float cpu_time = 1000.0f * (cpu_end - cpu_start) / CLOCKS_PER_SEC;
    printf("  时间: %.3f ms\n", cpu_time);
    
    // ===== GPU朴素版本 =====
    printf("\n[2] GPU朴素版本\n");
    dim3 blockSize(16, 16);
    dim3 gridSize((N + blockSize.x - 1) / blockSize.x,
                  (M + blockSize.y - 1) / blockSize.y);
    
    cudaEventRecord(start);
    matmulNaive<<<gridSize, blockSize>>>(d_A, d_B, d_C, M, K, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    cudaEventElapsedTime(&time, start, stop);
    
    cudaMemcpy(h_C_naive, d_C, M * N * sizeof(float), cudaMemcpyDeviceToHost);
    
    printf("  时间: %.3f ms\n", time);
    printf("  加速比: %.2fx\n", cpu_time / time);
    printf("  验证: %s\n", verifyResult(h_C_cpu, h_C_naive, M, N) ? "✓" : "✗");
    
    // ===== GPU共享内存版本 =====
    printf("\n[3] GPU共享内存版本\n");
    dim3 blockSize2(TILE_SIZE, TILE_SIZE);
    dim3 gridSize2((N + TILE_SIZE - 1) / TILE_SIZE,
                   (M + TILE_SIZE - 1) / TILE_SIZE);
    
    cudaEventRecord(start);
    matmulShared<<<gridSize2, blockSize2>>>(d_A, d_B, d_C, M, K, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    cudaEventElapsedTime(&time, start, stop);
    
    cudaMemcpy(h_C_shared, d_C, M * N * sizeof(float), cudaMemcpyDeviceToHost);
    
    printf("  时间: %.3f ms\n", time);
    printf("  加速比: %.2fx (vs CPU)\n", cpu_time / time);
    printf("  加速比: %.2fx (vs 朴素GPU)\n", cpu_time / time * time / cpu_time);
    printf("  验证: %s\n", verifyResult(h_C_cpu, h_C_shared, M, N) ? "✓" : "✗");
    
    // 性能分析
    printf("\n========================================\n");
    printf("性能分析\n");
    printf("========================================\n");
    
    // 计算GFLOPS
    double flops = 2.0 * M * K * N;  // 每个元素需要K次乘法和K次加法
    printf("总浮点运算: %.2f GFLOPS\n", flops / 1e9);
    printf("\nCPU版本:\n");
    printf("  性能: %.2f GFLOPS\n", flops / (cpu_time * 1e6));
    printf("\nGPU朴素版本:\n");
    printf("  性能: %.2f GFLOPS\n", flops / (time * 1e6));
    printf("\nGPU共享内存版本:\n");
    printf("  性能: %.2f GFLOPS\n", flops / (time * 1e6));
    
    // 清理
    cudaEventDestroy(start);
    cudaEventDestroy(stop);
    cudaFree(d_A);
    cudaFree(d_B);
    cudaFree(d_C);
    free(h_A);
    free(h_B);
    free(h_C_cpu);
    free(h_C_naive);
    free(h_C_shared);
}

int main() {
    benchmark();
    printf("\n========================================\n");
    printf("测试完成!\n");
    printf("========================================\n");
    return 0;
}
```

### 3.3 性能分析

#### 预期性能对比

在RTX 3090上，预期性能：

| 版本 | 时间 (ms) | 加速比 | GFLOPS |
|------|----------|--------|--------|
| CPU | ~5000 | 1x | ~0.4 |
| GPU朴素 | ~50 | 100x | ~40 |
| GPU共享内存 | ~5 | 1000x | ~400 |

**性能差距的原因**：

1. **CPU vs GPU朴素**：
   - CPU：顺序执行，优化编译器
   - GPU：大规模并行（1024x1024 = 1M线程）

2. **GPU朴素 vs GPU共享内存**：
   - 朴素版本：每次从全局内存读取A和B
   - 共享内存版本：数据复用，减少全局内存访问

#### 内存访问分析

**朴素版本**：
```
对于每个C[i][j]：
- 需要读取A的第i行：k次全局内存访问
- 需要读取B的第j列：k次全局内存访问
- 总计：2k次全局内存访问

优化空间：A和B的元素被重复读取
```

**共享内存版本**：
```
使用分块技术：
- 将矩阵分成32x32的小块
- 每个块加载到共享内存后复用
- 减少全局内存访问次数约32倍
```
