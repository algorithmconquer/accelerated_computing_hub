========================================
故障排查指南
========================================

如果您的输出与预期不符，请检查：

1. 编译错误
   - 确认CUDA Toolkit已正确安装
   - 检查环境变量PATH和LD_LIBRARY_PATH
   - 尝试降低CUDA架构版本：nvcc -arch=sm_70 ...

2. 运行时错误
   - 检查GPU驱动版本：nvidia-smi
   - 确认GPU显存足够（至少8GB）
   - 检查是否有其他程序占用GPU

3. 结果不正确
   - 检查边界条件处理
   - 确认__syncthreads()位置正确
   - 验证索引计算公式

4. 性能不佳
   - 确认使用了优化编译选项（-O3）
   - 检查Block大小设置（建议256-1024）
   - 使用nvprof工具分析性能瓶颈

5. 数值差异
   - 不同GPU型号性能差异正常
   - 浮点运算可能存在微小误差
   - 时间测量会有波动（多次运行取平均）

========================================
学习建议
========================================

1. 理解原理比记住代码更重要
   - 每个优化技巧都有硬件原理支撑
   - 思考为什么这样优化有效

2. 动手实验
   - 修改参数观察性能变化
   - 尝试不同的优化策略
   - 对比不同GPU型号的表现

3. 性能分析
   - 使用CUDA profiling工具
   - 分析内存访问模式
   - 计算理论性能上限

4. 循序渐进
   - 先保证正确性，再优化性能
   - 从简单问题开始，逐步增加复杂度
   - 理解每个示例后再进入下一个
